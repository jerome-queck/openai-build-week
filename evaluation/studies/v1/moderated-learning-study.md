# Moderated advanced-mathematics learning study v1

## Purpose and claim boundary

This formative protocol tests whether target learners can use Quick Study to understand and transfer an unfamiliar mathematical idea while recovering orientation and controlling context. It diagnoses the product; it is not powered or designed to establish causal educational impact. Report observations separately from benchmark reliability, and state that causal impact is unsupported unless a preregistered comparative study supplies that evidence.

## Participants and safeguards

- Recruit learners from the intended undergraduate through early-graduate range and record relevant prior coursework without assigning a fixed learner type.
- Obtain consent for observation and for any optional delayed follow-up separately. Participation, skipped questions, and declining the delayed task produce no negative Understanding Evidence.
- Use synthetic or study-provided sources. Do not expose participants' Personal Notes, credentials, local folders, or unrelated workspace material.
- Predeclare the target concept, one primary formative outcome, task order or counterbalancing plan, stopping rule, rubric, and material-harm boundary before the first session.

## Materials

- One unfamiliar source passage or proof with a pinned revision.
- Parallel unassisted pre-task and immediate post-task comprehension items.
- One unseen near-transfer item using the same concept or proof method with changed surface details.
- An optional delayed parallel near-transfer item scheduled for the participant-approved delay (seven days is the default proposal).
- A fixed candidate build and a completed environment record matching the quality-gate evidence.

No assessment item may appear in the tutoring context. The moderator must not coach the mathematical answer.

## Session procedure

1. Confirm consent, prior exposure, and the learner's interpretation of the task. Record baseline confidence from 0–100%.
2. Administer the unassisted pre-task. A blinded scorer later applies the same rubric used for post-task work.
3. Ask the learner to start a Learning Session, inspect or edit the Learning Goal, work with the supplied source, and explain the target idea in their own reasoning.
4. Observe navigation without steering: Source Anchor use, Context Used Receipt inspection, prerequisite navigation, backtracking, help required, recovery from one scripted interruption, and return to the exact task.
5. After the target task, collect the Paas 1–9 mental-effort item and a brief satisfaction comment. Treat the item as overall effort, not as a diagnosis of a particular load source.
6. Close model assistance. Administer the parallel comprehension item and then the unseen immediate near-transfer item. Collect confidence after each item.
7. Offer the delayed near-transfer follow-up. Record decline without penalty; if accepted, keep its unseen item and scoring blind until administration.
8. Ask the learner to identify the Learning Goal, current Return Point, unresolved issue, and next action. Record correctness and time-to-correct-resumption.

## Measures

| Dimension | Required record |
| --- | --- |
| Comprehension | Blind rubric score for meaning, assumptions, logical status, dependencies, high-level idea, and error localization or repair. |
| Reasoning quality | Evidence quoted from the learner's explanation; score strategy selection, warrant quality, and self-correction separately from final correctness. |
| Cognitive load | Paas 1–9 mental-effort response after the target task, plus the participant's optional explanation. |
| Navigation and recovery | Task success, elapsed time, backtracking count, help needed, Source Anchor and receipt inspection, interruption recovery, and Return Point accuracy. |
| Immediate near transfer | Blind score on the unseen parallel item, with confidence calibration recorded separately. |
| Optional delayed near transfer | Blind score, elapsed delay, and confidence; declining or missing the optional follow-up is not negative evidence. |

## Reporting

Use `session-instrument.md` for each participant, then aggregate without combining dimensions into one score. Report task failures and adverse observations, missing data, scorer disagreement, exceptions, and limitations. Engagement, completion, time in app, satisfaction, and artifact reuse are diagnostic only. Do not write “improves learning,” “causes transfer,” or equivalent causal language from this protocol.
