# Make agent access policy session-scoped

Each Learning Session will have a learner-selected Session Access Policy that bounds the local sources agents may read or use as model and research context and the agent tools that may act on them. The active policy remains visible and may be changed during the session through an explicit Access Elevation or restriction. Web corroboration through privacy-minimized Derived Research Queries is available under every policy.

Version one will offer three policies. Focused Access includes only material explicitly attached, pasted, highlighted, or selected for the current session. Workspace Access includes the Study Workspace's Managed Assets and Source Snapshots, supported files beneath its optional Primary Folder, and External Attachments explicitly added to that workspace. It excludes sibling folders, unrelated workspaces, and all other device content. Full Access allows broader local-file and agent-tool access during that session. Their web-search capability is the same; the difference is which local material may inform the work. Full Access does not authorize arbitrary modification or deletion of source files. Normal writes remain confined to app-managed storage and explicit user-selected artifact exports.

An access grant applies to its Learning Session, and changing policy does not erase the Session Record of work already performed under the earlier policy. The learner still selects an access policy for each session. An app-wide Access Confirmation Preference, enabled by default, determines whether selecting Full Access produces an additional confirmation; disabling it removes only that warning and does not select or carry Full Access into another session. Regardless of this preference, the active Session Access Policy remains visible throughout the session.

A new session launched inside an existing Study Workspace will begin with visible Workspace Access selected. A session started from a loose attachment, paste, or standalone question will begin with visible Focused Access selected. The learner may change either choice before or during work. Full Access is always off initially and is never inherited from another session.

Access Elevation remains off until the learner acts. The learner may select a broader policy proactively, or an agent may issue an Access Request explaining the reason, exact requested scope, and intended action. Approval changes only the current session; denial leaves the existing policy intact and requires the agent to continue within it or explain what cannot be completed. No agent may silently elevate itself or carry an elevated policy into another session.
