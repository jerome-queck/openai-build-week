# Preserve public history and canonicalize maintainer identity

Clarifold will preserve its existing public Git history rather than rewriting commits solely to replace Jerome Queck's historical `jqueck002@e.ntu.edu.sg` author email. A repository `.mailmap` will canonicalize that identity to Jerome Queck at `jeromequeck@jeromegroup.org` for normal Git attribution, and the maintainer's active Git configuration must use the Jerome Group address for future commits. The raw historical commits remain available, as they already are in public clones; `.mailmap` improves canonical presentation but is not represented as erasure.

A preliminary audit on 2026-07-22 found zero GitHub secret-scanning alerts, no secret-, credential-, environment-, or signing-file names in commit history, and no unexpectedly large historical objects in the largest-object sample. This evidence is reassuring but not a substitute for a dedicated content scan. The public-repository implementation sweep must run and record a full-history secret scan, audit unusually large objects and contributor identities, and distinguish real secrets or private learner material from false positives.

History rewriting is reserved for a confirmed serious exposure such as a credential, private learner material, or similarly sensitive content. If such material is found, containment starts by revoking or rotating the affected secret and assessing copies and downstream impact; a coordinated history rewrite follows only when its reduction in continuing exposure justifies invalidating commit identifiers and disrupting existing clones. Cosmetic renaming, removal of obsolete Build Week references, and ordinary documentation cleanup do not justify rewriting history.
