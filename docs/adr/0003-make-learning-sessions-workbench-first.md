# Make Learning Sessions workbench-first

Learning Sessions will be organized around a Mathematical Workbench whose primary object is the learner's PDF, proof, exercise, note, diagram, or question; conversation is a contextual companion rather than the primary surface. This deliberately rejects a chat-first shell because the product's differentiation depends on direct manipulation, source anchoring, reasoning state, and durable artifacts, while preserving open-ended conversation when it is useful.

Teaching Moves requested from a Source Anchor will first appear in a Contextual Inspector linked visibly to that anchor. Short explanations and questions remain in the inspector; substantial diagrams, proof walkthroughs, or other durable outputs may be expanded or pinned into the main canvas as Learning Artifacts. Closing the inspector leaves only a subtle Anchor Marker on the Source Layer, preventing the source from accumulating an inline message feed.

Follow-up interaction about the same anchor will revise one current synthesized Teaching Card rather than append learner and agent messages chronologically. Prior revisions remain recoverable from the Session Record. A genuinely different route, such as a geometric rather than algebraic explanation, becomes a separately named Teaching Variant so alternatives remain comparable without being confused with incremental refinement.

Open-ended conversation will remain available through a universal Ask Bar rather than a persistent transcript panel. When a Source Anchor is active, it supplies the immediate context; otherwise the bar uses the Learning Goal and visible session context. A question creates or revises a structured Question Card. Learner and Teaching Orchestrator interaction remains in the Session Record for recovery; internal Specialist Agent messages and tool execution remain only in the Agent Work Log.

Before submission, the Ask Bar will expose its working context as editable Context Chips. Every chip must state what it represents through a type label, human-readable identity, and relevant location or preview, such as `Source · Algebra notes · p. 12`, `Selection · “Let M be…”`, or `Goal · Understand exactness`. Icons and colors may reinforce these distinctions but cannot replace the text. The learner can add or remove context without opening a separate settings surface. Only a small set of primary chips remains visible; excess items collapse behind a clear `+N context` control, with the exact threshold deferred to interface testing.

After submission, agents may retrieve additional relevant material within the active Session Access Policy. Each resulting Teaching Card will provide an expandable Context Used Receipt listing the context and source locations actually used. Retrieval outside the active policy still requires Access Elevation.
