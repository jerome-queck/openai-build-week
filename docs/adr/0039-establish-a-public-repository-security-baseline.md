# Establish a public-repository security baseline

Clarifold's public repository uses GitHub private vulnerability reporting, Dependabot alerts and security updates, secret scanning, repository push protection, CodeQL default setup, and protected `main`. The `main` rule requires pull requests, the strict `verify` status check, linear history, and resolved review conversations; it blocks force-pushes and branch deletion. It requires zero approving reviews because Jerome Queck is the sole maintainer, and administrator bypass remains available only for documented emergency recovery. Merged branches continue to be deleted automatically and GitHub Discussions remain disabled.

CodeQL default setup analyzes GitHub Actions and JavaScript/TypeScript under the `remote_and_local` threat model on a weekly schedule. GitHub's first automatic Swift autobuild failed because the native helper needs an explicit build; GitHub adjusted default setup to exclude Swift and the resulting Actions and JavaScript/TypeScript setup run completed successfully. Swift security analysis therefore remains a separately tracked custom-analysis requirement rather than a silently failing default check.

The repository adds `.github/dependabot.yml` with grouped monthly npm and GitHub Actions version updates and grouped security updates; this file becomes active when merged. The 2026-07-21 initial baseline identified unresolved development-dependency advisories and CodeQL alerts spanning command construction, filesystem paths, dynamic dispatch, and workflow permissions. Parent issue #81 retains the dated aggregate intake snapshot already published during planning; the current per-alert inventory, locations, source-to-sink detail, and any confirmed sensitive findings remain in private triage records. A focused security-triage and remediation change must validate or dismiss each finding with evidence, update affected dependencies, add Swift analysis, and reach the approved security gate before the Clarifold public-repository cutover.

Security automation discovers work but never self-approves it. The mandatory route is scanner alert, security triage, validated private finding or safely worded public ticket, targeted security fix, focused reproducer or regression test, full `npm run verify`, security diff scan, ordinary code review, protected pull request and CI, merge, and evidence-backed alert or advisory closure. Dependabot pull requests never auto-merge. `/implement` may execute an approved security ticket but does not replace finding triage or security validation. The repository adapter, coding standards, and pull-request template must add this security-impact routing and declaration without modifying vendored Matt skills. Confirmed sensitive vulnerabilities remain in private advisories until coordinated disclosure is safe.
