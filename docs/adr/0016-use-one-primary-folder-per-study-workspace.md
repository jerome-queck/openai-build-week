# Use one primary folder per Study Workspace

A Study Workspace may link one optional Primary Folder. Workspace Access can inspect relevant supported files beneath that directory, subject to the active Session Access Policy, while keeping source modification and deletion restricted. A workspace without a linked folder remains valid and can operate entirely from Managed Assets.

Files dropped directly into the app may be retained as Managed Assets, and occasional sources outside the Primary Folder may be added as individual External Attachments. Version one will not support an arbitrary set of independent folder roots; that complexity can be introduced later if real workspace usage demonstrates the need.
