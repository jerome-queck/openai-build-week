# Keep local work available without model access

Loss of network connectivity, Codex authentication, subscription capacity, API quota, or the local Codex Runtime will not make the desktop product unusable. In Local Working Mode, learners retain full access to local sources, Session Records, annotations, Source Indexes, Learning Trails, Learning Artifacts, search, editing, and export. An installed Verifier Runtime may continue performing local Lean checks.

Model-dependent Teaching Moves and online Source Corroboration will be marked unavailable rather than hanging, fabricating results, or being confused with local capability. Learners may save Pending Questions in the Ask Bar, but the app will not submit them automatically when access returns; the learner reviews and sends them explicitly.
