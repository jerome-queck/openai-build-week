# Name the product Clarifold

The public product and repository brand is Clarifold, presented as "the advanced mathematics learning workbench" with the central promise "Turn the feeling of understanding into evidence." The name combines clarity and unfolding, with a secondary mathematical association to manifolds; it replaces Quick Study as the app name so Quick Study can remain the distinct built-in workspace for loose, unfiled Learning Sessions. The repository and package slug will be `clarifold`, the macOS application and executable will be `Clarifold`, and the new macOS bundle identifier will be `org.jeromegroup.clarifold`.

This is an application-identity and data migration, not a cosmetic string replacement. The first Clarifold launch must safely preserve an existing Quick Study beta data directory: stage a copy into the new Clarifold location, validate it before activation, leave the old directory intact as a rollback source, and never silently delete or merge data. An explicit development or test data-directory override must not trigger migration of a person's default application data. If both old and new locations already contain data, the app must stop automatic migration and offer a truthful recovery path rather than choosing or merging silently. Existing Linked Sources remain externally owned and are not copied; their durable paths and identities must survive migration. The built-in Quick Study workspace name and its durable domain identifiers remain unchanged. `CLARIFOLD_*` becomes the canonical environment-variable namespace through one central configuration adapter; only the user-facing `QUICK_STUDY_DATA_DIR` is accepted for one transitional Clarifold beta with deprecation, while internal and test-only names are changed immediately and stored identifiers remain stable where migration risk requires it. Packaging, CI, release evidence, application metadata, repository links, documentation, screenshots, runtime client identity, compatibility identifiers, and installed-app tests must be updated together and verified from a packaged Clarifold artifact. The old Quick Study data must remain usable for rollback until the user explicitly removes it in a future governed flow.

Preliminary screening found no obvious education-software collision but did find unrelated pharmaceutical, physical-mirror, and historical audio uses, so formal trademark clearance remains a future gate before material commercial launch or trademark investment.
