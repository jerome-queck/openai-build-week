# Preserve imported material as a non-editable source layer

Imported and linked textbooks, PDFs, images, and notes will appear as a non-editable Source Layer inside the product. The app will not rewrite the learner's original files when highlighting, explaining, correcting, or adapting their mathematics. Learner annotations, selected diagram regions, Teaching Moves, verification evidence, and Trail Items will attach to precise Source Anchors instead.

For a Linked Source, the Source Layer is a read-only view of the currently accessible Source Revision, not a hidden historical copy. If the external file changes or disappears, the prior fingerprint, anchors, and derived session records remain, but the app will state honestly when the exact old content is unavailable. Exact historical content is retained only when the learner explicitly creates a Source Snapshot in app-managed storage.

Any simplified, expanded, corrected, or pedagogically reorganized proof will be stored as a separate editable and versioned Reformulated Proof. It will preserve links to the Source Anchors it interprets so the learner can compare the reformulation with the original and distinguish source content from agent-generated work.

Selecting text, an equation, or a boxed diagram region will create or target a Source Anchor and open a compact Selection Palette rather than immediately dispatching an agent. Its primary actions are explain or unpack, ask about this, annotate, and add to the Learning Trail. Context-specific actions such as visualize, verify, compare approaches, or create a Prerequisite Branch remain available without crowding the default palette.
