# License public releases for noncommercial use

The project is presently owned by its sole individual owner, Jerome Queck, and will carry the notice `Copyright © 2026 Jerome Queck`. A root `LICENSE.md` must contain the unmodified PolyForm Noncommercial 1.0.0 (`PolyForm-Noncommercial-1.0.0`) terms for the owner's original software; the README must explain the boundary in plain language without replacing or altering the controlling text. Once that legal baseline is implemented, people may use, modify, and distribute the owner's software for permitted noncommercial purposes, while every commercial use requires a separate written permission from the owner. Commercial Permission is requested through `licensing@jeromegroup.org`, decided case by case, and may be free, paid, conditional, or refused. A request should identify the requester and organization, intended product or service, expected users, proposed use or modification, distribution method, and whether revenue is involved; sending a request grants no rights by itself. This preserves commercial control during the product's protective phase while still enabling public learning and noncommercial experimentation. During that phase, public issues, proposals, security reports, and testing feedback are welcome, but outside code contributions are not accepted; accepting them later requires contributor terms that preserve the owner's commercial-relicensing authority.

Third-party components retain their own licences and notices. `THIRD_PARTY_NOTICES.md` must cover the components actually shipped, and a deterministic packaged-artifact audit must include Electron and Chromium notices, bundled Lean and mathlib, native helpers, and transitive runtime dependencies rather than checking only top-level `package.json`. The implemented CI gate must fail on unknown or disallowed shipped-component licences and missing required notice text. A machine-readable SBOM joins the future signed/notarized public-release workflow but is not required for every development build. Clarifold must be described as source available rather than open source while this decision applies.

The canonical machine-readable license identifier will be `PolyForm-Noncommercial-1.0.0`, including in `package.json`; the package remains marked `private` to prevent accidental npm publication. The root `LICENSE.md` must contain only the exact official terms. A root `NOTICE` must carry the required Jerome Queck copyright and Clarifold brand notice, while `THIRD_PARTY_NOTICES.md` will be the authority for shipped third-party attributions. The packaged application must include all three documents. README and About Clarifold may explain and link to them in plain language but do not replace or modify the controlling terms. GitHub detection is useful metadata, not authority and not evidence that Clarifold is open source.

Accepting the first collaborative contribution is a hard intellectual-property gate. No code, design, icon, documentation, or substantial mathematical-content contribution from another person may merge merely because it was submitted through an approved pull request or is offered under the repository's outbound license. Before accepting it, Clarifold must classify the contributor relationship, decide whether ownership remains with Jerome Queck or moves to a Singapore entity, and use written Singapore-appropriate terms that either assign the relevant copyright to that owner or grant sufficiently broad, irrevocable, sublicensable commercial rights. If retaining a single owner remains the priority, assignment is preferred while preserving attribution. A DCO sign-off alone does not transfer ownership. The agreement and professional review are deferred until real collaboration is imminent, but collaboration cannot begin first and repair the rights chain afterward.
