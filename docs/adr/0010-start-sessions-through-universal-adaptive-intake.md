# Start sessions through universal adaptive intake

The dashboard will use one prominent Session Intake for typed questions, pasted mathematics, dropped PDFs or images, and selected Study Workspace files. Learners will not be required to classify their input as a proof, exercise, concept, or notes before beginning. The Teaching Orchestrator will interpret the material and produce an editable Session Proposal containing the likely Learning Goal, working scope, and initial teaching direction.

This intake preserves open-ended use without reducing the product to a chat interface. A compact Session Access Policy control remains available at launch, and the app asks a clarifying question only when the missing information materially affects the proposed session.

Session Confirmation will be consequence-based rather than mandatory. A clear, small request begins immediately while keeping its inferred Learning Goal visible and editable. Long, ambiguous, or multi-topic material pauses on the concise Session Proposal only when a mistaken interpretation would waste meaningful effort; the learner can then confirm or revise the goal, scope, or starting direction.
