# Remove Build Week-specific governance

Jerome Queck has withdrawn the project from OpenAI Build Week. Clarifold therefore has no Devpost submission, judging, deadline, `/feedback`, or event-specific public-repository obligation. The Clarifold public-repository migration must delete `docs/openai-build-week.md`; remove the Build Week section and `Codex-Feedback-Session` requirement from `AGENTS.md`; remove the corresponding Devpost trailer instructions from `CONTRIBUTING.md`; replace event-specific reasoning in research and platform documentation with product-owned reasoning; and remove or migrate repository URLs that use `openai-build-week`. Useful AI provenance remains under [ADR-0041](0041-record-ai-assistance-without-ai-coauthorship.md): neutral `Assisted-by` and authentic session trailers replace AI `Co-authored-by` and event-specific feedback trailers.

This decision does not remove generally useful engineering rigor that happened to be developed during the event. Packaged-app verification, deterministic quality gates, truthful learning-evidence boundaries, release provenance, recovery testing, and candidate-bound checks remain Clarifold product and release contracts unless separately superseded. Historical Git commits are not rewritten merely to erase the event context.
