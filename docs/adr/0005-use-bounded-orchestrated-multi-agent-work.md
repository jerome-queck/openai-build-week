# Use bounded orchestrated multi-agent work

The product will default to one active Teaching Agent under a Teaching Orchestrator, which preserves one coherent learning experience and dispatches task-scoped Specialist Agents only when a concrete trigger predicts enough value to justify their cost and coordination overhead. Specialist work will run sequentially whenever one result can affect the next Agent Brief; parallel fan-out is reserved for genuinely independent tasks, such as exploring alternative solution routes or independently verifying a result. Each specialist receives a minimal Agent Brief rather than the full session history and returns structured results with provenance into shared session state; peer-to-peer agent chatter and unbounded fan-out are rejected. Agent Budgets will select agent count, concurrency, model, reasoning effort, tools, tokens, and latency per task, with maximum reasoning treated as exceptional rather than the default.

Agent Budget selection will be automatic by default. Learners may bias a Learning Session through a simple Faster, Balanced, or Deeper Reasoning Preference, while optional advanced Runtime Overrides expose exact model and effort choices that the active Codex Runtime supports. Maximum reasoning will not be dispatched silently for routine Teaching Moves, even when the session preference is Deeper.
