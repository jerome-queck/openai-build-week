# Separate personal notes from tutor feedback

Source annotations will have two explicit purposes. Add note creates a Personal Note retained locally and excluded from ordinary teaching context. Tell tutor creates durable Tutor Feedback that may revise the current Teaching Card, guide later Teaching Moves, and inform future Learning Artifact synthesis. The interface will label the distinction at creation and allow the learner to change an annotation's purpose later.

This separation prevents private working notes from silently becoming model input while ensuring that teaching feedback survives beyond one response. Tutor Feedback remains anchored to the source or artifact region it concerns so later agents can use it without reconstructing its meaning from a raw transcript.

An app-wide Personal Note Synthesis Preference will determine whether Personal Notes may be supplied to the model specifically during final artifact generation. It is enabled by default and can be disabled by the learner in settings. This preference does not make Personal Notes available to routine Teaching Moves or change them into Tutor Feedback.

When that preference is enabled, artifact synthesis will preserve and include the learner's original Personal Note verbatim. It may also create a linked Note Interpretation that improves grammar, clarity, and organization for use in the artifact's surrounding prose. The interpretation never overwrites or silently substitutes for the learner's original words, so both authorship and pedagogical intent remain inspectable.
